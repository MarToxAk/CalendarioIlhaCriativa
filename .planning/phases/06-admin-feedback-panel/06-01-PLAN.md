## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→admin namespace | Admin autenticado envia requests GET com params client_id e status |
| form PATCH→admin_reply | Admin envia texto livre via campo textarea para update no controller |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | GET /admin sem autenticação | mitigate | Admin::BaseController herda require_authentication — qualquer request não autenticado redireciona para /session/new; verificado nos testes com sign_in_as |
| T-06-02 | Information Disclosure | params[:client_id] no filtro do dashboard | accept | Admin tem acesso total a todos os clientes por design — não é cross-client leak; o dashboard é o painel centralizado do admin (CONTEXT.md: "admin vê dados de outro cliente via params... por design") |
| T-06-03 | Tampering | XSS via conteúdo de admin_reply renderizado na view | mitigate | Rails escapa HTML por padrão em <%= %>; proibido usar raw() com admin_reply; campo está no namespace admin (não visível ao cliente, D-10) |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | Esta fase não instala pacotes externos — N/A |
</threat_model>
