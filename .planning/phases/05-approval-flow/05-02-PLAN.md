| Scope de segurança | @client.artes.find levanta RecordNotFound para artes de outros clientes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Elevation of Privilege | Client::ResponsesController#set_arte | mitigate | @client.artes.find(params[:arte_id]) — nunca Arte.find direto; rescue RecordNotFound redireciona para calendário sem vazar informação |
| T-05-02-02 | Tampering | ApprovalResponse#create | mitigate | Validator arte_must_be_pending (server-side) rejeita re-aprovação de arte não aprovável; botões ocultos são UX, não barreira |
| T-05-02-03 | Tampering | Strong params decision | mitigate | response_params usa permit(:decision, :comment) estritamente; status da arte nunca é parâmetro aceito |
| T-05-02-04 | Spoofing | CSRF em form de aprovação | mitigate | form_with inclui authenticity_token automaticamente; csrf_meta_tags no layout client.html.erb |
| T-05-02-05 | Tampering | decision value via hidden_field | mitigate | Enum validation no model — decision fora do enum {:approved, :change_requested} causa ArgumentError; server-side barreira dupla |
