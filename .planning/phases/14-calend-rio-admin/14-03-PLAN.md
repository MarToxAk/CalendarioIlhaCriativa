## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Turbo Frame request (GET /admin/calendar?month=...) | Request parcial de Turbo Frame herda a sessão autenticada — a autenticação é verificada pelo before_action do Admin::BaseController em cada request |
| link_to admin_arte_path(arte) no chip | Link de chip aponta para rota admin autenticada — não expõe URLs de clientes, apenas rota admin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05 | Spoofing | Turbo Frame request GET /admin/calendar?month= | mitigate | Cada request (inclusive partial Turbo Frame) passa pelo before_action :require_authentication do Admin::BaseController — sessão inválida redireciona [VERIFIED: padrão Phase 13] |
| T-14-06 | Tampering | Parâmetro month= em links de setas (ERB) | accept | @prev_month e @next_month são gerados server-side pelo controller com strftime — não interpolam input do usuário na view diretamente |
| T-14-07 | Information Disclosure | Iniciais do cliente visíveis no calendário | accept | Calendário é admin-only; iniciais (2 chars) não constituem dados sensíveis; acesso já protegido por autenticação |
| T-14-SC | Tampering | npm/pip/cargo installs | accept | Esta fase não instala nenhum pacote novo — sem risco de supply chain |
