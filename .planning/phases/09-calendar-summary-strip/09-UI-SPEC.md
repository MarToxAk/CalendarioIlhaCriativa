## Component Inventory

This phase introduces **one new UI element** — the Summary Strip — with no new components extracted to partials.

### Summary Strip

| Property | Value |
|----------|-------|
| Container | Inline `<div>` in `index.html.erb`, between nav header and calendar grid |
| Visibility | Hidden when `@summary[:total] == 0` (D-06) — use `<% if @summary[:total] > 0 %>` guard |
| Layout | `flex flex-wrap gap-2 mb-4 justify-center` or `justify-start` |
| Partial | None — inline in `index.html.erb` (D-10) |
| Update strategy | SSR redirect — no JS, no Turbo Stream (D-09) |

### Chip anatomy

Each chip is a `<span>` with:
- `px-3 py-1 rounded-full text-xs font-medium` base classes
- Status-specific background + text color classes (see Color table above)
- Count number rendered as `font-semibold` inline, followed by label text

Chip structure (ERB pseudo-code):
```
<span class="px-3 py-1 rounded-full text-xs font-medium bg-{color}-100 text-{color}-{weight}">
  <span class="font-semibold"><%= @summary[:total] %></span> total
</span>
```

Chips rendered in fixed order: **total → aprovadas → pendentes → pediu alteração**

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Strip visible | Only when `@summary[:total] > 0` |
| Strip hidden | When month has no artes (full `<% if %>` block suppressed — no empty wrapper rendered) |
| After approval | Client is redirected back to calendar; controller recalculates `@summary`; strip reflects new counts automatically (SSR) |
| Mobile layout | `flex-wrap: wrap` — chips wrap to next line when viewport is narrow; labels are NOT abbreviated (D-08) |
| Touch targets | Chips are display-only (not interactive); no touch target minimum required |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Strip ARIA | Wrap strip in `<div role="status" aria-label="Resumo do mês">` so screen readers announce the counts |
| Chip color | Color alone does not convey status — count number + text label present on every chip |
| Reduced motion | Existing `@media (prefers-reduced-motion)` in `application.css` handles; no animations added |
| Color contrast | green-800 on green-100 = 5.3:1 (WCAG AA pass); amber-800 on amber-100 = 5.1:1 (pass); red-800 on red-100 = 5.5:1 (pass); slate-700 on slate-100 = 5.9:1 (pass) |
