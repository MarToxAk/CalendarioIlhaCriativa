<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL /c/:token → Client lookup | O token no URL é a primeira camada de autenticação do cliente |
| client.authenticate(password) | Senha do cliente cruza a camada de autenticação |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | access_token enumeração | mitigate | Token base58 24-char: espaço de busca 58^24 ≈ 10^42; Rack::Attack no GET /c/:token (Plano 04) limita enumeração |
| T-03-02 | Elevation of Privilege | Cross-client Arte access | mitigate | Queries SEMPRE escopadas via `@client.artes.find(params[:id])` — nunca `Arte.find()` diretamente (reforçado no Plano 05) |
| T-03-03 | Information Disclosure | password_digest na DB | accept | bcrypt hash; nunca exposto via API; has_secure_password não serializa a coluna por padrão |
| T-03-04 | Tampering | Enum reordenação | mitigate | Enums definidos com hash explícito `{ instagram: 0 }` — nunca array; NUNCA reordenar chaves existentes |
| T-03-05 | Elevation of Privilege | arte_must_be_pending bypass | mitigate | Validação on: :create no model; status só muda via ApprovalResponse#sync_arte_status — não via update direto |

</threat_model>
