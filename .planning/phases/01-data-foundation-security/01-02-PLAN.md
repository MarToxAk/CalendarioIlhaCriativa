<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /session | Credenciais de email + senha do admin cruzam esta fronteira |
| Cookie assinado → Session lookup | O cookie contém session_id assinado — se comprometido, atacante pode impersonar admin |
| db/seeds.rb → produção | Senha hardcoded no seeds — não usar em produção sem override via ENV |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | SessionsController#create (admin login) | mitigate | bcrypt via has_secure_password (fator de custo adaptativo); `authenticate_by` ou `authenticate` — nunca comparar digest manualmente |
| T-02-02 | Tampering | Cookie de sessão admin | mitigate | Rails usa cookies.signed (HMAC-SHA256); Session model no DB permite revogação instantânea via destroy |
| T-02-03 | Information Disclosure | db/seeds.rb senha hardcoded | mitigate | seeds.rb é apenas para dev; em produção usar ENV var: `u.password = ENV.fetch("ADMIN_PASSWORD")` — anotar no seeds.rb |
| T-02-04 | Elevation of Privilege | Admin routes sem autenticação | mitigate | Admin::BaseController tem `before_action :require_authentication` — todas as rotas admin herdam este before_action |
| T-02-05 | Denial of Service | POST /session brute-force (admin login) | mitigate | Rack::Attack throttle "admin/login_by_ip" 5 req/60s — configurado no Plano 04 |
| T-02-SC | Tampering | bundle install (gems) | accept | Gems auditados no RESEARCH.md; bcrypt 388M downloads, autores Ruby core |

</threat_model>
