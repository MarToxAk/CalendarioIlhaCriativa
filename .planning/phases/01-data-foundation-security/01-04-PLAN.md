<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Rack middleware | Rack::Attack intercepta antes do router — proteção de camada mais externa |
| POST /c/:token/session | Ponto de entrada para brute-force de senha do cliente |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Denial of Service | POST /c/:token/session brute-force | mitigate | Throttle 1: 5 tentativas/20s por token; Throttle 2: 10/60s por IP como fallback |
| T-04-02 | Information Disclosure | Enumeração de tokens via GET /c/:token | mitigate | Throttle 4: 20 GET/60s por IP; token base58 24-char torna força bruta computacionalmente inviável |
| T-04-03 | Denial of Service | POST /session brute-force admin | mitigate | Throttle 3: 5 req/60s por IP no /session (login admin) |
| T-04-04 | Spoofing | Bypass de throttle via IP spoofing | accept | X-Forwarded-For pode ser falsificado; o discriminator principal é o token (não o IP) para o Throttle 1 — IP spoofing não bypassa proteção principal |
| T-04-05 | Denial of Service | Atacante bloqueia IP legítimo usando token de outro | accept | Throttle por token protege o token do atacante; o cliente legítimo não é bloqueado pelo token de outro cliente |

</threat_model>
