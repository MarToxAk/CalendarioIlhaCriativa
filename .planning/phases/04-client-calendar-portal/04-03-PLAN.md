<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Client::ArtesController#show | params[:id] é input do usuário (tentativa de IDOR cross-client) |
| view → external_url | URL de origem externa renderizada como href — potencial open redirect ou phishing |
| view → ActiveStorage blob | URL assinada ou proxy path expõe conteúdo de arquivo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Elevation of Privilege | Client::ArtesController — IDOR via params[:id] | mitigate | `@client.artes.find(params[:id])` — ActiveRecord::RecordNotFound se arte pertence a outro cliente → redirect; teste explícito de cross-client no artes_controller_test.rb |
| T-04-03-02 | Tampering | external_url renderizado como href | mitigate | `target="_blank" rel="noopener noreferrer"` — sem redirect server-side; o browser do cliente assume o risco do destino externo |
| T-04-03-03 | Spoofing | iframe de external_url | mitigate | Decisão D-12 proíbe iframe; apenas botão com _blank; view NÃO deve conter tag iframe |
| T-04-03-04 | Information Disclosure | ActiveStorage blob URL para vídeo | mitigate | rails_service_blob_proxy_path (sem URL assinada exposta, sem expiração) — apenas cliente autenticado chega à view |
| T-04-03-05 | Information Disclosure | ActiveStorage blob URL para imagem | accept | rails_blob_path (URL assinada, 5min expiry) — adequado para imagem; URL expira antes de ser utilizável por terceiros |
| T-04-03-SC | Tampering | npm/pip/cargo installs | accept | Nenhuma instalação de pacotes neste plano |
</threat_model>
