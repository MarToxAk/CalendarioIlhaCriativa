<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebSocket handshake → connection.rb | Token de URL (/cable?token=abc123) e cookie de sessão chegam como input não-confiável |
| connection.rb → banco de dados | params[:token] é usado como argumento de find_by — passa pelo ORM parameterizado |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01-01 | Elevation of Privilege | connection.rb#connect | mitigate | reject_unauthorized_connection como última instrução — sem conexões anônimas (D-04); implementado no connect: set_current_user \|\| set_current_client \|\| reject_unauthorized_connection |
| T-17-01-02 | Information Disclosure | set_current_client | mitigate | Client.find_by(access_token: token, active: true) — condição active: true rejeita clientes inativos na connection (D-03); sem feedback sobre qual condição falhou (timing-safe) |
| T-17-01-03 | Information Disclosure | params[:token] brute-force | accept | Token de 24 chars via has_secure_token (SecureRandom) — 144 bits de entropia; Rack::Attack rate limiting já instalado no projeto cobre /cable endpoint |
| T-17-01-04 | Tampering | ActiveRecord injection via params[:token] | accept | ActiveRecord parameteriza automaticamente os argumentos de find_by — sem risco de SQL injection; token é lido como string e passado para ORM |
| T-17-SC | Tampering | npm/pip/cargo installs | accept | Esta fase não instala pacotes externos |
</threat_model>
