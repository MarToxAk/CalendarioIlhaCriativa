## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → turbo-cable-stream-source | Elemento DOM gerado server-side com stream name assinado por Turbo.signed_stream_verifier — cliente não consegue forjar |
| sidebar → banco de dados | Arte.where(status: :change_requested).count é query COUNT server-side; sem input do usuário |
| DOM → toast_controller.js | Toasts na Phase 17 são estrutura estática (sem conteúdo dinâmico externo); Phase 18 deve fazer html_escape do conteúdo broadcast |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-03-01 | Tampering | turbo_stream_from stream name | mitigate | Stream name é assinado por Turbo.signed_stream_verifier com chave secreta do Rails — cliente não consegue assinar stream de outro usuário; guard "if current_user" previne erro se layout renderizar sem user |
| T-17-03-02 | Information Disclosure | Arte.where(status: :change_requested).count | accept | Badge expõe apenas um número COUNT — não expõe IDs, nomes de artes, ou dados de clientes; aceitável para admin autenticado |
| T-17-03-03 | Tampering | XSS via toast content (futuro) | accept | Phase 17 não appenda conteúdo dinâmico via toast_controller; Phase 18 deve garantir html_escape no template Turbo Stream server-side antes de broadcast; anotar para Phase 18 |
| T-17-03-04 | Denial of Service | timer leak em toast_controller | mitigate | clearTimeout(this._timerId) no disconnect() garante que timers de toasts removidos antes de 5s não continuam executando; _enforceLimit() limita a 3 toasts no DOM |
| T-17-SC | Tampering | npm/pip/cargo installs | accept | Esta fase não instala pacotes externos; toast_controller.js usa apenas @hotwired/stimulus já instalado via importmap |
</threat_model>
