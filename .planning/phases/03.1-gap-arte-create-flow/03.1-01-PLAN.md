<threat_model>
## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| Browser admin → Rails | Input de formulário não confiável: client_id (hidden field pode ser adulterado pelo admin) |
| URL params → Controller | params[:client_id] em GET new vem da URL — qualquer admin autenticado pode fornecer qualquer valor |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3.1-01 | Tampering | `Admin::ArtesController#create` — `arte_params` inclui `:client_id` | accept | Admin autenticado pode associar arte a qualquer cliente: comportamento esperado. `Admin::BaseController` já exige autenticação via `before_action :require_authentication`. Cross-client leak só é risco no portal do cliente (já mitigado nas fases anteriores com escopo por token). |
| T-3.1-02 | Tampering | `Admin::ClientsController#show` — `@artes = @client.artes` | accept | `@client` é carregado via `set_client` usando `params[:id]` do URL, e o admin é autenticado. A query já é escopada ao cliente correto — sem risco de cross-client. |
| T-3.1-03 | Information Disclosure | `clients/show.html.erb` — lista de artes do cliente | accept | View acessível apenas para admin autenticado (herança `Admin::BaseController`). Sem exposição a usuários não autenticados. |
| T-3.1-SC | Tampering | npm/pip/cargo installs | accept | Nenhum pacote novo instalado neste plano — apenas alterações de views, controller e testes Ruby. |
</threat_model>
