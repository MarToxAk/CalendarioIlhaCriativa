# Phase 03.1 Plan 01: Gap Arte Create Flow Summary

## One-liner

Fechamento de 4 gaps de produção no fluxo de criação de artes: hidden_field client_id, fix de duplicate class no media toggle, link Artes na sidebar e card de artes no show do cliente.

## What Was Built

### Task 1: Corrigir _form.html.erb (commit c1b6da0)

Dois bugs críticos corrigidos no formulário de artes:

**Bug 1 — client_id nunca propagado no POST:** O `Arte.new(client: @client)` no controller preenchia o atributo no objeto, mas nenhum hidden_field enviava o valor no submit. Toda arte criada chegava com `client_id nil`, falhando a validação do model. Adicionado `<%= f.hidden_field :client_id %>` antes do botão de submit.

**Bug 2 — Duplicate class attribute no media toggle:** As divs dos campos `uploadField` e `linkField` usavam `class="mb-4"` estático + `<%= "class='hidden'" unless ... %>` dinâmico, gerando dois atributos `class` no mesmo elemento HTML. O browser descarta o segundo atributo — logo o campo que deveria ficar oculto aparecia sempre. Corrigido com expressão ternária única: `class="<%= arte.media_file.attached? ? 'mb-4' : 'mb-4 hidden' %>"`.

### Task 2: Adicionar link Artes na sidebar (commit 4391127)

Inserido `{ label: "Artes", path: admin_artes_path }` no array `nav_items` da sidebar, posicionado após "Clientes" e antes de "Calendário". A rota `admin_artes_path` já existia (`resources :artes` no namespace admin).

### Task 3: Seção Artes no clients/show + @artes no controller (commit 51f0f84)

**Controller:** `Admin::ClientsController#show` agora carrega `@artes = @client.artes.order(scheduled_on: :desc)`. Variável `@artes_with_responses` preservada (card de histórico de aprovações intacto).

**View:** Card "Artes" adicionado após o card "Informações", antes do "Histórico de Aprovações". Contém:
- Botão "Nova Arte" linkando para `new_admin_arte_path(client_id: @client.id)`
- Lista iterando `@artes` com título, data (`strftime`), plataforma, status e link "Ver"
- Mensagem "Nenhuma arte cadastrada." quando `@artes.empty?`

### Task 4: Testes GET new com client_id (commit 62adfd8)

Adicionados 2 testes em `Admin::ArtesControllerTest`:
- `"should get new with client_id pre-filled"`: asserta 200 e que o body contém o valor do client_id
- `"should get new without client_id"`: asserta 200 sem parâmetros

Suite completa: 13 testes, 0 failures, 0 errors, 0 skips.
