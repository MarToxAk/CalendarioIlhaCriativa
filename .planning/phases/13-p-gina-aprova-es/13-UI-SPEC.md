## Spacing Scale

Declarado em `--spacing-*` no CSS e aplicado via classes Tailwind:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`) | Gaps de ícone, padding interno de badges |
| sm | 8px (`p-2`) | Padding compacto de células, gap entre filtros |
| md | 16px (`p-4`) | Padding padrão de células de tabela (`px-4 py-3`) |
| lg | 24px (`mb-6`) | Separação entre barra de filtros e tabela |
| xl | 32px (`mb-8`) | Separação entre seções de página |
| 2xl | 48px (`py-16`) | Padding vertical do estado vazio |
| 3xl | 64px | Spacing de nível de página (não utilizado nesta fase) |

Exceptions: touch target mínimo de 32px (`h-8`) nos botões de ação "Ver arte" — conforme padrão Phase 11.

## Layout e Estrutura da Página

### Estrutura geral

```
[Sidebar verde] | [Área de conteúdo — bg-[#F9FAFB] flex-1 p-8]
                  ├── H1 "Aprovações"
                  ├── Barra de filtros (FORA do turbo-frame)
                  └── <turbo-frame id="approvals-content">
                        ├── Tabela paginada OU
                        └── Estado vazio
```

### Barra de filtros

Replicar exatamente o padrão de `app/views/admin/dashboard/index.html.erb`:

```
form_with url: admin_approvals_path, method: :get,
          data: { turbo_frame: "approvals-content" },
          class: "flex items-center gap-3 mb-6"
```

- **Dropdown 1 — Cliente:** `[["Todos os clientes", ""]] + clientes`
  - Classes: `h-9 px-3 border border-gray-200 rounded-lg text-sm text-slate-700 bg-white`
- **Dropdown 2 — Decisão:** `[["Todas as decisões", ""], ["Aprovado", "approved"], ["Pediu Alteração", "change_requested"]]`
  - Classes: mesmas do dropdown de cliente
- **Botão Filtrar:** `h-9 px-4 bg-[#0F7949] hover:bg-[#0a5c37] text-white text-sm font-semibold rounded-lg transition-colors cursor-pointer`

### Tabela desktop (hidden sm:block)

Replicar o padrão de `app/views/admin/artes/index.html.erb`:

```
<div class="bg-white rounded-xl border border-gray-200 shadow-card overflow-hidden hidden sm:block">
  <table aria-label="Lista de aprovações" class="w-full">
    <caption class="sr-only">Respostas de aprovação — clique em Ver arte para ver os detalhes</caption>
    <thead class="bg-gray-50 border-b border-gray-200">
```

**Colunas (source: 13-CONTEXT.md `<specifics>`):**

| # | Header | Classe th | Conteúdo td |
|---|--------|-----------|-------------|
| 1 | CLIENTE | `text-left` | `approval_response.arte.client.name` — `text-sm text-slate-900` |
| 2 | ARTE | `text-left` | `approval_response.arte.title` (ou "Sem título") — `text-sm text-slate-900` |
| 3 | DECISÃO | `text-left` | Badge `_decision_badge` partial — ver spec de badge abaixo |
| 4 | DATA DA RESPOSTA | `text-left` | `responded_at.strftime("%d/%m/%Y")` — `text-sm text-slate-600` |
| 5 | COMENTÁRIO | `text-left` | Truncado em 80 chars; vazio se nil — `text-sm text-slate-500 italic` quando nil: `"—"` |
| 6 | AÇÕES | `w-[48px]` + `sr-only` span | Botão "Ver arte" |

**Cada linha:**
```
<tr class="hover:bg-gray-50 transition-colors border-b border-gray-100 last:border-0">
```

**Botão "Ver arte":**
```
link_to "Ver arte", admin_arte_path(arte),
  class: "h-8 px-3 border border-gray-200 rounded-lg text-xs text-slate-600 hover:bg-gray-50 transition-colors"
```

### Lista de cards mobile (block sm:hidden)

Um card por resposta, replicando o padrão mobile de artes:

```
<div class="block sm:hidden space-y-3">
```

Cada card (`block bg-white rounded-xl border border-gray-200 shadow-card px-4 py-3`):
- Linha 1: nome do cliente (semibold) + badge de decisão
- Linha 2: título da arte + data em `text-xs text-slate-500`
- Linha 3 (se comentário): truncado em 60 chars, `text-xs text-slate-400 italic`
- Chevron direito + link para `admin_arte_path(arte)`

### Badge de decisão (novo partial: `_decision_badge`)

Criar `app/views/admin/approvals/_decision_badge.html.erb` baseado no padrão de `_status_badge.html.erb`:

```
approved:
  bg-[#F0FDF4] text-[#14A958] border-[#14A958]/20  → "Aprovado"
change_requested:
  bg-[#FEF2F2] text-[#EE3537] border-[#EE3537]/20  → "Pediu Alteração"
```

Estrutura do span:
```
<span class="inline-flex items-center gap-1 px-2 py-1 rounded-full text-xs font-medium border {badge_classes}">
  <span aria-hidden="true">●</span> {label}
</span>
```

### Paginação

Posição: abaixo da tabela, dentro do `turbo-frame`.
Usar helper `pagy_nav(@pagy)` com classes existentes do projeto.
Alinhamento: `flex justify-center mt-4`.

## Estados de Interação

### Estado de filtro ativo

- Quando `params[:client_id]` ou `params[:decision]` estiver preenchido, o `selected:` do dropdown reflete o valor.
- O turbo-frame `approvals-content` é substituído sem recarregar sidebar/filtros.
- Sem indicador visual extra de "filtro ativo" nesta fase — os dropdowns já mostram o valor selecionado.

### Estado de hover nas linhas

- `hover:bg-gray-50 transition-colors` — padrão Phase 11, já definido.

### Estado de loading (Turbo)

- Turbo adiciona `[aria-busy="true"]` no frame automaticamente durante requests.
- Nenhum spinner customizado necessário — comportamento padrão do Turbo é suficiente.

### Estado vazio

- Ícone SVG centralizado (estilo `text-slate-300 mb-4` `width="48" height="48"`) — inline heroicon adequado: `inbox` ou `check-circle` (outline).
- Heading `text-lg font-semibold text-slate-900 mb-1`.
- Body `text-sm text-slate-500 mb-6`.
- Sem botão CTA no estado vazio (nenhuma ação de criação disponível nesta página).

## Acessibilidade

- `<table>` com `aria-label="Lista de aprovações"`.
- `<caption class="sr-only">` descrevendo ação disponível.
- Coluna de ações: `<th><span class="sr-only">Ações</span></th>`.
- Badge de decisão: texto legível dentro do span (não só ícone).
- Focus ring: `box-shadow: 0 0 0 3px rgba(15,121,73,0.30)` já definido globalmente em `:focus-visible`.
- `prefers-reduced-motion`: já coberto globalmente em `application.css`.
- Dropdowns de filtro: `<select>` nativo — acessível por padrão.
