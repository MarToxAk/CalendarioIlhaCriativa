<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Admin controller | Todas as actions de ClientsController exigem sessão admin autenticada via before_action :require_authentication herdado de BaseController |
| Admin form → DB | Strong params em client_params filtram campos permitidos; password e password_plain em branco na edição são rejeitados (D-10) |
| password_plain column | Dado sensível em texto puro na tabela clients — acessível apenas por admin autenticado; não exposto em views públicas |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Admin::ClientsController | mitigate | Herdado: before_action :require_authentication em BaseController bloqueia acesso não autenticado; redireciona para /session/new |
| T-02-02 | Tampering | client_params strong params | mitigate | params.require(:client).permit(:name, :password, :password_plain, :active) — campos não listados ignorados |
| T-02-03 | Tampering | rotate_token action | mitigate | Rota POST member — CSRF token obrigatório (Turbo/Rails padrão); only admin pode acessar |
| T-02-04 | Information Disclosure | password_plain coluna | accept | Acessível apenas por admin autenticado via HTTPS; tradeoff intencional documentado em D-08/D-09 |
| T-02-05 | Elevation of Privilege | Admin routes sem role check | accept | Sistema tem um único admin (sem multi-role em v1); require_authentication é suficiente |
| T-02-SC | Tampering | npm/gem installs | mitigate | Sem novos pacotes neste plano — apenas arquivos Ruby/ERB/JS internos |
</threat_model>
