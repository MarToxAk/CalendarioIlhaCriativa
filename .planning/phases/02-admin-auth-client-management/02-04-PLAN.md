    `<% hidden_fields&.each { |key, value| concat hidden_field_tag(key, value) } %>`

    Para suportar hashes aninhados como `{ client: { active: false } }` (Rails espera `client[active]` como nome do campo), a iteração deve fazer flatten do hash.    
    ```
    hidden_fields (estrutura esperada): { client: { active: false } }
    nome do hidden field gerado: "client[active]", value: "false"
    ```

    Implementar o loop dentro do `form_with do |f|` ANTES do `f.submit`:
    - Se `hidden_fields` não é nil: iterar `hidden_fields.each do |outer_key, outer_value|`
      - Se `outer_value` é um Hash: iterar `outer_value.each do |inner_key, inner_value|` e chamar `concat hidden_field_tag("#{outer_key}[#{inner_key}]", inner_value)`
      - Se `outer_value` não é Hash: chamar `concat hidden_field_tag(outer_key, outer_value)`

    ```
    unless @client.active?
      render plain: "Acesso bloqueado", status: :forbidden
    end
    ```

    Atenção: o guard deve ser inserido antes do `rescue ActiveRecord::RecordNotFound` — o rescue deve permanecer cobrindo apenas o find_by!. A estrutura correta após a edição:
    1. `@client = Client.find_by!(access_token: params[:token])`
    2. `unless @client.active? ... render ... end`
    3. `rescue ActiveRecord::RecordNotFound`
    4. `  render plain: "Link inválido", status: :not_found`

    ```
    end
    ```
