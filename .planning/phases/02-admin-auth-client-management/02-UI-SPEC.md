## Interaction Patterns

### Padrão 1 — Copiar (link e senha)

**Trigger:** Clicar em "Copiar link" ou "Copiar senha"

**Comportamento:**
1. JavaScript executa `navigator.clipboard.writeText(valor)`
2. Botão muda imediatamente: ícone clipboard → ícone check verde + texto "Copiado!"
3. Após 2 segundos: botão retorna ao estado original (`"📋 Copiar link"` / `"📋 Copiar senha"`)
4. Nenhum toast separado — o feedback é inline no botão

**Estilo do botão de copiar (estados):**

Estado padrão:
```
h-8 inline-flex items-center gap-2 rounded-md px-3
text-xs font-medium text-slate-600
bg-white border border-gray-200
hover:bg-gray-50 hover:border-gray-300
transition-all 150ms ease
```

Estado copiado (2s):
```
text-success (#14A958)
border-success/30
bg-success-bg (#F0FDF4)
```

Fallback (clipboard API não disponível):
- Seleciona o texto do input (`input.select()`)
- Mostra: "Selecione e copie" inline no botão por 3s

**Markup do botão:**
```html
<button
  type="button"
  data-copy-target="link-input"
  aria-label="Copiar link de acesso"
  aria-live="polite"
>
  <!-- ícone clipboard 14px -->
  Copiar link
</button>
```

---

### Padrão 2 — Desativar cliente

**Trigger:** Clicar em "Desativar cliente" no topo da tela de detalhes

**Comportamento:**
1. Abre um modal de confirmação (não redireciona, não usa alert() do browser)
2. Modal exige que o admin confirme antes de executar a ação
3. Submit do modal: POST/PATCH para `admin_client_path` com `active: false`
4. Após sucesso: redirect para `/admin/clients/:id` com flash de sucesso
5. Badge de status atualiza para "Inativo"
6. Botão "Desativar cliente" na barra de ações substitui por "Reativar cliente" (bg success)

**Modal de confirmação — Desativar:**
```
┌─────────────────────────────────────────────┐
│  Desativar cliente                     [✕]  │
│  ─────────────────────────────────────────  │
│  Desativar Loja da Maria bloqueará o        │
│  acesso ao portal imediatamente.            │
│  O cliente não conseguirá acessar o         │
│  calendário até ser reativado.              │
│                                             │
│    [Manter cliente ativo]  [Desativar cliente]  │
└─────────────────────────────────────────────┘
```

- Botão "Manter cliente ativo": Secondary — fecha o modal
- Botão "Desativar cliente": Danger (bg #EE3537) — executa a ação

**Comportamento — Reativar cliente:**
- Trigger: botão "Reativar cliente" na barra de ações
- Sem modal de confirmação (reativar não é destrutivo)
- POST/PATCH direto com `active: true`
- Flash de sucesso: "Loja da Maria reativada. O cliente pode acessar o portal novamente."

### Padrão 4 — Modal compartilhado

Todos os modais de confirmação usam o mesmo padrão estrutural:

```
Overlay: fixed inset-0 bg-black/50 z-50 flex items-center justify-center
Modal:   bg-white rounded-xl shadow-lg p-6 max-w-md w-full mx-4
         animation: fadeIn 200ms ease
Header:  font-semibold text-slate-900 text-base + botão [✕] top-right
Body:    text-sm text-slate-600 leading-relaxed mt-2
Footer:  flex justify-end gap-3 mt-6
```

**Teclado:**
- `Escape` fecha o modal (equivale ao botão de cancelar contextual)
- `Tab` cicla apenas dentro do modal (focus trap)
- `Enter` no botão de ação destrutiva = executa (não necessita Enter para cancelar)
- Foco inicial: no botão de cancelar contextual (para prevenir ativação acidental)

**ARIA do modal:**
```html
<div
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-title"
  aria-describedby="modal-desc"
>
  <h2 id="modal-title">Desativar cliente</h2>
  <p id="modal-desc">...</p>
</div>
```

### Navegação por teclado — tabela

- `Tab`: percorre links de nome de cliente, badges (não focáveis), datas (não focáveis), botões "…"
- `Enter` no botão "…": abre dropdown
- `Arrow Down/Up` no dropdown aberto: navega entre itens de menu
- `Escape` no dropdown aberto: fecha, foco retorna ao botão "…"

### Navegação por teclado — formulários (new/edit)

- `Tab`: Nome → Senha → toggle olho → "Voltar sem salvar" → Submit
- `Enter` em qualquer campo: submete o formulário
- `Escape`: sem efeito (usuário usa botão "Voltar sem salvar")

### Navegação por teclado — modal

- Foco inicial: botão de cancelar contextual ("Manter cliente ativo" / "Manter link atual") — previne ativação acidental de ação destrutiva
- `Tab`: cicla apenas entre elementos dentro do modal (focus trap)
- `Escape`: fecha o modal = equivale ao botão de cancelar contextual
- `Enter` no botão de ação: executa

### Anúncios de live region

```html
<!-- Feedback de copiar -->
<span aria-live="polite" class="sr-only" id="copy-status"></span>
<!-- JS injeta "Link copiado para a área de transferência" no clique -->

<!-- Flash messages -->

</div>
```

## Responsiveness

### Mobile (< 768px)

**Lista de clientes — tabela → stacked cards:**

A tabela não cabe em telas pequenas. Em mobile, substituir por lista de cards empilhados:

```
┌─────────────────────────────────┐
│  Loja da Maria      [Ativo]     │
│  Criado em 20/05/26     [>]     │
└─────────────────────────────────┘
┌─────────────────────────────────┐
│  Boutique Coral     [Inativo]   │
│  Criado em 15/05/26     [>]     │
└─────────────────────────────────┘
```

- Cada card: `bg-white rounded-xl border border-gray-200 shadow-card px-4 py-3`
- Nome: text-sm font-semibold text-slate-900
- Badge: inline após o nome
- Seta [>]: `text-slate-400`, toque leva ao detalhe
- Sem menu "…" em mobile — ações ficam na tela de detalhe

Botão "+ Novo cliente": full-width em mobile, abaixo do título.

**Formulário (new/edit) em mobile:**
- Card: sem borda arredondada extrema, `rounded-lg`, `mx-4`
- Inputs: `height: 48px` (área de toque)
- Botões: stacked ("Voltar sem salvar" acima, Submit abaixo), ambos full-width

**Detalhe em mobile:**
- Barra de ações (Editar cliente / Desativar cliente): move para o final da página, abaixo dos cards
- Ou: botões empilhados full-width no final
- Card de dados de acesso: full-width
- Input readonly URL: `overflow-hidden text-ellipsis` — URL muito longa
- Botão copiar: full-width, `h-10`

**Modal em mobile:**
- `mx-4 max-w-full` — ocupa quase toda a largura
- Não usa bottom sheet — mantém modal centralizado

### Tablet (768px–1023px)

- Sidebar colapsada (64px, só ícones) — herdado de Phase 1
- Tabela mantida, mas coluna "Criado em" pode ser ocultada se necessário (`hidden md:table-cell`)

### Desktop (≥ 1024px)

- Layout padrão: sidebar 240px + conteúdo
- Tabela completa com todas as colunas
- Formulário: card `max-w-lg` à esquerda, não centralizado no main area
