<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → rotate_token action | POST member requer CSRF token (Turbo/Rails); autenticação admin via BaseController |
| Browser → update (deactivate) | PATCH com active:false via form_with dentro do modal; CSRF protegido |
| password_plain no show | Exibido apenas para admin autenticado; campo readonly — não editável pelo browser diretamente |
| client_portal_url | Gerado server-side a partir de access_token do DB; não editável pelo admin na tela show |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Tampering | rotate_token POST | mitigate | CSRF token obrigatório (form_with padrão Rails); somente admin autenticado acessa — dupla proteção |
| T-02-10 | Spoofing | Modal confirmation | accept | Modal é client-side UX, não segurança — a proteção real é o CSRF token no form do modal |
| T-02-11 | Information Disclosure | password_plain no DOM | accept | Exibido apenas na tela show acessível somente por admin autenticado; tradeoff intencional (D-08/D-09) |
| T-02-12 | Denial of Service | Rotação de token | accept | Admin pode rotacionar indefinidamente; sem rate-limit em v1 (volume baixo, 10-30 clientes) |
| T-02-13 | Tampering | copy_controller — clipboard | accept | navigator.clipboard só funciona em HTTPS/localhost; produção usa HTTPS — sem risco |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | Sem novos pacotes — apenas views ERB e JS sem dependências externas |
</threat_model>
