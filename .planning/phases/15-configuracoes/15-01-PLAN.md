# Wave 1 — Migration + Route + Sidebar

## Context
Esta wave cria a fundação da Fase 15:
1. Migração `agency_name` na tabela `users`
2. Rota `resource :settings` no namespace admin
3. Sidebar com link wired e nome da agência dinâmico
4. Scaffold de testes

## Tasks

### Task 1 — Migração agency_name

Criar migração:
```
rails generate migration AddAgencyNameToUsers agency_name:string
```

A migração deve:
- Adicionar coluna `agency_name :string, null: false, default: "Ilha Criativa"`
- Executar `db:migrate`

Verificar que `db/schema.rb` contém a coluna.

### Task 2 — User model: validação

Editar `app/models/user.rb`:
```ruby
class User < ApplicationRecord
  has_secure_password
  has_many :sessions, dependent: :destroy

  normalizes :email_address, with: ->(e) { e.strip.downcase }

  validates :agency_name, presence: true, length: { maximum: 100 }
end
```

### Task 3 — Rotas

Editar `config/routes.rb`, dentro do bloco `namespace :admin do`:

```ruby
resource :settings, only: [:show] do
  patch :update_password, on: :member
  patch :update_agency,   on: :member
end
```

Isso gera:
- `admin_settings_path` → GET /admin/settings
- `update_password_admin_settings_path` → PATCH /admin/settings/update_password
- `update_agency_admin_settings_path` → PATCH /admin/settings/update_agency

### Task 4 — Sidebar

Editar `app/views/admin/shared/_sidebar.html.erb`:

1. Substituir `path: "#"` de "Configurações" por `path: admin_settings_path`
2. No branding (div `px-4 py-5 border-b`), substituir o texto hardcoded "Ilha Criativa" por `Current.user&.agency_name || "Ilha Criativa"` e remover a segunda linha hardcoded "by Bom Custo" (que não tem coluna correspondente no modelo):

```erb
<div class="px-4 py-5 border-b border-white/20">
  <span class="text-white font-semibold text-sm block"><%= Current.user&.agency_name || "Ilha Criativa" %></span>
</div>
```

3. O helper `current_page?` não funciona corretamente com `resource :settings` (singular) — para evitar active state quebrado, usar `request.path.start_with?(admin_settings_path)` para o item Configurações:

```ruby
nav_items = [
  { label: "Dashboard",     path: admin_root_path },
  { label: "Aprovações",    path: admin_approvals_path },
  { label: "Clientes",      path: admin_clients_path },
  { label: "Artes",         path: admin_artes_path },
  { label: "Calendário",    path: admin_calendar_index_path },
  { label: "Configurações", path: admin_settings_path },
]
```

O `current_page?` compara exatamente o path, o que é adequado para `/admin/settings` (a rota show).
