<objective>
Corrigir CR-03 (media_source descartado pelo servidor) e dois warnings relacionados (WR-01 e WR-02) identificados no 07-REVIEW.md:

CR-03: O form envia `arte[media_source]=upload` ou `arte[media_source]=link` via radio buttons, mas `media_source` não está em `arte_params` e é descartado silenciosamente pelo Rails. No update, se uma arte tem `media_file` anexado e o admin seleciona "Link externo", o arquivo pré-existente não é desanexado — a validação `only_one_media_source` rejeita o save com "Use arquivo OU link externo, não ambos". O admin vê um erro sem causa aparente.

WR-01: Em uma arte nova (sem media_file e sem external_url), nenhum radio fica selecionado. O Stimulus não exibe nenhum campo de mídia, deixando o form sem opção visível de upload ou link.

WR-02: O botão "Editar" na view show só aparece quando `@arte.pending?`, mas `check_editable` permite edição também para `revised?` e `change_requested?`. O admin não encontra o botão na UI para artes revisadas ou com pedido de alteração.

Purpose: Honrar o radio button de media_source no controller para que a exclusão mútua entre arquivo e link funcione corretamente no update; pré-selecionar upload como padrão para artes novas; expor o botão Editar para todos os status editáveis.
Output: artes_controller.rb com lógica de media_source em update e create; _form.html.erb com radio upload pré-selecionado; show.html.erb com condição de Editar expandida.
</objective>
