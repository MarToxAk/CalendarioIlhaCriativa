## What Was Built

### CR-03: media_source Honrado no Controller

O método `update` foi reescrito em três etapas:

1. `@arte.assign_attributes(arte_params)` — atribui atributos sem salvar
2. `case params.dig(:arte, :media_source)` — processa o radio button:
   - `when "link"`: `@arte.media_file.purge_later if @arte.media_file.attached?`
   - `when "upload"`: `@arte.external_url = nil`
3. `if @arte.save` — salva e redireciona ou renderiza `:edit`

O método `create` recebeu o mesmo processamento de media_source após `Arte.new(arte_params)`:

```ruby
case params.dig(:arte, :media_source)
when "upload"
  @arte.external_url = nil
end
```

`media_source` não entra em `arte_params` — lido diretamente via `params.dig` para evitar mass assignment. Nenhum `attr_accessor :media_source` adicionado ao model.

### WR-01: Radio Upload Pré-selecionado

Em `_form.html.erb`, linha 37, o `checked:` do radio `:upload` foi alterado de:

```
checked: arte.media_file.attached?
```

para:

```
checked: arte.media_file.attached? || arte.external_url.blank?
```

Para artes novas (sem arquivo e sem external_url), o radio "Upload de arquivo" fica selecionado por padrão, tornando o campo de arquivo visível ao carregar o form. O radio `:link` (`checked: arte.external_url.present?`) permanece inalterado.

### WR-02: Botão Editar Expandido

Em `show.html.erb`, linha 17, a condição do link "Editar" foi alterada de:

```erb
if @arte.pending?
```

para:

```erb
if @arte.pending? || @arte.revised? || @arte.change_requested?
```

Agora alinhado com a lógica de `check_editable` no controller.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Assertion T2 ajustada para comportamento real do Rails após save falho**
- **Found during:** Task 1 (GREEN phase)
- **Issue:** O teste T2 verificava `@arte.reload.external_url.blank?` após um save que falha. No Rails, quando `save` retorna `false`, o banco não é alterado — a mudança existe apenas no objeto em memória. A assertion estava incorreta.
- **Fix:** Alterado para verificar que a resposta é 422 E que o erro exibido é `media_source_present` ("Precisa de arquivo ou link externo"), não `only_one_media_source` ("Use arquivo OU link externo"). Essa distinção confirma que `external_url` foi zerado antes do save — se não fosse zerado, ambos (`media_file` e `external_url`) estariam presentes e o erro seria `only_one_media_source`.
- **Files modified:** `test/controllers/admin/artes_controller_test.rb`
- **Commit:** a2e15a0
