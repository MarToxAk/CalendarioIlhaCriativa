## What Was Built

### CR-01: Guard em set_client

O método `set_client` foi reescrito em duas etapas:

1. `return unless params[:client_id].present?` — preserva o comportamento original quando `client_id` está ausente (retorna imediatamente sem atribuir `@client`)
2. `unless @client` após `find_by` — quando `client_id` está presente mas não encontra cliente válido, redireciona para `admin_artes_path` com `alert: "Cliente não encontrado."`

Adicionalmente, `index` foi incluído no `before_action :set_client` para que o filtro de `client_id` na página de listagem também seja protegido contra IDs inválidos.

### CR-02: Verificação de Retorno Booleano em destroy

O método `destroy` foi alterado de chamada incondicional para condicional:

```ruby
if @arte.destroy
  redirect_to admin_artes_path, notice: "Arte excluída com sucesso."
else
  redirect_to admin_arte_path(@arte), alert: "Não foi possível excluir a arte."
end
```

### Hook de Test no Modelo Arte

Para testar o branch `else` do destroy sem Mocha, foi adicionado um `before_destroy` test-only condicionado a `Rails.env.test?`:

```ruby
if Rails.env.test?
  class << self
    attr_accessor :test_block_destroy
  end
  self.test_block_destroy = false

  before_destroy :__test_block_destroy_hook__
  def __test_block_destroy_hook__
    throw :abort if self.class.test_block_destroy
  end
end
```

Isso permite que integration tests controlem o retorno de `@arte.destroy` sem dependências externas.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical Functionality] index adicionado ao before_action :set_client**
- **Found during:** Task 1 (RED phase)
- **Issue:** O plano especificava "GET /admin/artes com params client_id inválido redireciona com alert" como Behavior 1. A ação `index` não estava no `before_action :set_client`, portanto o guard nunca seria chamado para requisições ao index
- **Fix:** Adicionado `index` ao `before_action :set_client, only: %i[index new create show edit update destroy mark_revised]`
- **Files modified:** `app/controllers/admin/artes_controller.rb`
- **Commit:** 9d0b814

**2. [Rule 2 - Missing Critical Functionality] Hook de test para simular destroy false**
- **Found during:** Task 2 (RED phase)
- **Issue:** O plano requeria testar `@arte.destroy` retornando `false`, mas o projeto não tem Mocha. Sem um mecanismo de stub, é impossível testar o branch `else` do CR-02
- **Fix:** Adicionado `before_destroy` hook test-only em `app/models/arte.rb` condicionado a `Rails.env.test?`, usando `Arte.test_block_destroy` como flag de classe
- **Files modified:** `app/models/arte.rb`
- **Commit:** 9fa3645

**Nota técnica:** Durante a investigação do hook de test, tentativas de usar `class_eval` no `test_helper.rb` ou `prepend` de módulos no nível do teste falharam porque o autoloader do Zeitwerk/Rails carrega as classes após os arquivos de test setup. Mover o hook para o próprio modelo (`arte.rb`) com guard `Rails.env.test?` resolve o problema definitivamente.
