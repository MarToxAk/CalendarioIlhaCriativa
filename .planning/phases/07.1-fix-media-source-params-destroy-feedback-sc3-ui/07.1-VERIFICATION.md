### Human Verification Required

#### 1. Consistência SSR do campo de upload em artes novas

**Test:** Abrir `/admin/artes/new` no browser com JavaScript desabilitado (ou usar curl/lynx). Verificar estado inicial do formulário.

**Expected:** O radio "Upload de arquivo" aparece selecionado E o campo de arquivo está visível ao mesmo tempo. Não deve haver estado contraditório onde o radio está checked mas o campo de arquivo está oculto.

**Why human:** A linha 37 do `_form.html.erb` usa `arte.media_file.attached? || arte.external_url.blank?` para `checked:` do radio, mas a linha 41 usa apenas `arte.media_file.attached?` para a classe CSS do div que envolve o campo de arquivo. Para uma arte nova, `external_url.blank?` é true mas `media_file.attached?` é false, resultando em radio selecionado + campo hidden no HTML gerado pelo servidor. O Stimulus corrige isso em `connect()`, mas a inconsistência SSR existe. Só é possível confirmar visualmente se isso gera problema real na UX.
