Reescrever app/models/approval_response.rb seguindo a API correta de broadcast em AR models.

Diagnóstico do problema atual: O código chama turbo_stream.append(...), turbo_stream.replace(...) e turbo_stream.prepend(...) dentro do model. O método turbo_stream é definido em Turbo::Streams::TurboStreamsTagBuilder (via controller) e Turbo::StreamsHelper (via view). Turbo::Broadcastable inclui instance methods como broadcast_append_to, broadcast_replace_to etc. que internamente chamam Turbo::StreamsChannel.broadcast_*_to — mas NÃO define o método turbo_stream.

Solução: Construir cada turbo-stream tag como string HTML usando ApplicationController.render para renderizar o HTML do conteúdo do partial e turbo_stream_action_tag para embrulhá-lo na tag turbo-stream. O Turbo::Broadcastable expande Turbo::Streams::ActionHelper via include no módulo de instância, portanto turbo_stream_action_tag está disponível como método de instância em ApprovalResponse (verificar via grep no broadcastable.rb). O conteúdo final é uma string HTML concatenada enviada via AdminNotificationsChannel.broadcast_to(admin, content) — o ActionCable transmite a string e o Turbo no frontend processa cada turbo-stream tag.

Passos exatos para o método broadcasts_to_admin reescrito:

      - Renderiza o HTML do conteúdo: `html = ApplicationController.render(partial: partial, locals: locals, formats: [:html])`
      - Monta a tag: `turbo_stream_action_tag(action, target: target, template: html)`
      - Retorna a string da tag.

Nota sobre turbo_stream_action_tag no model: Turbo::Broadcastable faz `extend Turbo::Streams::ActionHelper` no ClassMethods e o include no Concern. Verificar se turbo_stream_action_tag está disponível como método de instância. Se NÃO estiver disponível como instância method, chamar diretamente como `Turbo::StreamsChannel.send(:turbo_stream_action_tag, ...)` (privado mas acessível via send) OU incluir o ActionHelper explicitamente. Alternativa mais simples: construir a string da turbo-stream tag manualmente usando ERB/String interpolation — as tags são simples: `<turbo-stream action="ACAO" target="TARGET"><template>HTML</template></turbo-stream>`.

Usar a alternativa mais simples e testável: construir a string de cada tag diretamente:

```
def turbo_stream_tag(action, target, template_html = "")

end
```

Isso elimina a dependência de turbo_stream_action_tag no model. O Turbo no frontend processa essas tags independentemente do como foram geradas.
