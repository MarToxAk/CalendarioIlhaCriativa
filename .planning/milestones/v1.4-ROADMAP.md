## Milestone Summary

**Key Decisions:**

- Cor por cliente derivada de `client.id % 8` — determinístico, sem coluna de cor no model Client
- Pagy habilitado via include em base_controller/application_helper (não via initializer) — escopo circunscrito ao namespace admin
- Decision badge usa hash config (não case/when) — mais conciso para 2 casos
- resources :calendar (plural) em vez de resource singular — garante helper _index_path e action index
- form_with url: (não model:) nas settings — não há ActiveRecord object sendo editado diretamente
- BrazilianHolidays em app/lib/ (autoloaded) — não em app/models ou lib/ standalone
- Holiday span colocado fora do if/else do número do dia — dia atual + feriado exibe ambos
- Rack::Attack.cache.store.clear no setup de testes — previne flakiness do rate limiter entre testes

**Issues Resolved:**

- Sidebar links "Aprovações" e "Calendário" apontavam para `#` desde v1.0 — corrigidos nas fases 13 e 14
- Rack::Attack MemoryStore acumulava throttle hits entre testes — fix cirúrgico no setup
- title=client.name adicionado ao chip do calendário admin — resolve assert_includes client.name + melhora acessibilidade

**Issues Deferred:**

- Validações visuais (HUMAN-UAT + VERIFICATION) das fases 13 e 14 — deferred como padrão do projeto (todos os scores automatizados são máximos: 15/15 e 9/9)

**Technical Debt Incurred:**

- app/lib/ foi criado neste milestone — requer server restart para reconhecer novos arquivos (documentado na fase 16)
- Cobertura de feriados é 2025-2027 — requer expansão anual
