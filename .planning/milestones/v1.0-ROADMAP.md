**Status:** ✅ SHIPPED 2026-05-27

**Total Plans:** 23

---

## Overview

O v1.0 entrega o sistema completo de aprovação de artes de social media: fundação de dados e segurança, painel do admin com CRUD de clientes e artes, portal do cliente com calendário mensal e fluxo de aprovação, e painel de feedback do admin. Ao final da Fase 6, o admin consegue gerenciar clientes e artes, e o cliente consegue aprovar ou pedir alteração em cada arte pelo link, sem conta.

## Milestone Summary

### Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Rails 8.1.3 + auth generator (sem Devise) | Escolha do usuário + gen nativo elimina dependência | ✓ Bom — auth simples e controlável |
| Tailwind v4 CSS-native (sem tailwind.config.js) | Versão instalada pela gem tailwindcss-rails 4.x | ✓ Bom — design tokens via @theme funcionaram |
| Gems em vendor/bundle | bot user não está no grupo rvm | ✓ Necessário — sem alternativa |
| PostgreSQL em 192.168.3.203 (remoto) | Servidor existente da infra Chatwoot | ✓ Funcionou — .env com credenciais |
| scheduled_on :date (não datetime) | Evita problemas de timezone na visualização do calendário | ✓ Bom — artes no dia correto |
| Queries sempre escopadas por @client | Previne cross-client data leak | ✓ Seguro — Rule aplicada em todos os controllers do portal |
| client.persisted? no _form | Distingue new vs edit sem variáveis extras | ✓ Elegante |
| Locale pt-BR manual (sem rails-i18n) | Evita gem pesada para poucas strings | ✓ Suficiente para v1 |
| Turbo Frame para filtros do dashboard | Filtros sem recarregar a página completa | ✓ UX responsiva |
| has_many :approval_responses (não has_one) | Permite histórico completo de decisões por arte | ✓ Correto — re-aprovação funciona |
| Phase 2.1 inserida (INSERTED) | password_plain stale após update — detectado em auditoria | ✓ Fechado — CLIE-04 satisfeito |
| Phase 3.1 inserida (INSERTED) | form sem client_id — blocker de criação de artes | ✓ Fechado — ARTE-01→07 satisfeitos |

- PAIN-05 bug: admin_reply bloqueado por check_editable — Phase 06-04 corrigiu para change_requested?/revised?
