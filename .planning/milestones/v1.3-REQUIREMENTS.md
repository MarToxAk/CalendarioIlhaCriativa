# Requirements — v1.3 Arte UI Polish

## Milestone Goal

Estilizar completamente as páginas de artes do admin usando Tailwind direto, eliminando os placeholders `form-input`/`btn` sem CSS definido e tornando o sistema visualmente consistente.

---

## v1.3 Requirements

### Form de Artes

- [x] **FORM-01**: Admin vê os campos do form de artes (text, textarea, date, url, file, select) estilizados com classes Tailwind — border, focus ring verde, height uniforme, placeholder visível
- [x] **FORM-02**: Admin vê os botões do form de artes (Criar/Atualizar e Cancelar) estilizados — verde para submit, neutro para cancelar
- [x] **FORM-03**: Radio buttons de "Tipo de mídia" têm layout horizontal com gap e labels legíveis

### Páginas new/edit de Artes

- [x] **PAGE-01**: Página "Nova Arte" envolve o form em card branco com link de voltar (padrão igual ao de clientes)
- [x] **PAGE-02**: Página "Editar Arte" envolve o form em card branco com link de voltar com nome da arte

### Arte Index

- [x] **IDX-01**: Tabela de artes tem thead com headers estilizados, td com padding e hover nas rows
- [x] **IDX-02**: Botão "Nova Arte" e link "Ver" na index têm estilo visível

### Arte Show

- [x] **SHOW-01**: Botões de ação no show (Editar, Excluir, Marcar Revisada, Voltar) têm estilos visíveis e semânticos (vermelho para excluir)

### Dashboard

- [x] **DASH-01**: Link "Ver" no painel de respostas tem estilo visível
