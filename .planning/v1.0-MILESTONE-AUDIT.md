## Critical Gaps

### Gap 1 — ARTE-01: Formulário de criação de artes sem `client_id` (BLOCKER)

**Requisitos afetados:** ARTE-01, ARTE-02, ARTE-03, ARTE-04, ARTE-05, ARTE-06, ARTE-07

**Problema:** `Admin::ArtesController#new` recebe `@client = Client.find_by(id: params[:client_id])`. O link "Nova Arte" em `admin/artes/index.html.erb` não passa `client_id`. O formulário `_form.html.erb` não tem campo `client_id`. Toda submissão falha com "Client can't be blank" — erro que o usuário não consegue resolver.

**Sintoma observado pelo usuário:** Não existe página no painel admin para criar artes e adicioná-las ao calendário do cliente.

**Fix:** Duas opções:
- **Opção A (simples):** Adicionar `<%= f.hidden_field :client_id %>` no `_form.html.erb` e atualizar os links de "Nova Arte" para passar `?client_id=X` (e.g., a partir da tela do cliente).
- **Opção B (flexível):** Adicionar `<%= f.select :client_id, Client.order(:name).map { |c| [c.name, c.id] } %>` ao formulário para que o admin escolha o cliente na criação.

---

### Gap 2 — CLIE-04: `password_plain` não atualizado após edição de senha (BLOCKER)

**Requisito afetado:** CLIE-04

**Problema:** `Admin::ClientsController#update` não sincroniza `password_plain` quando a senha muda. Apenas o `create` action tem o merge `password_plain: password`. Após editar a senha do cliente, a tela de detalhes exibe a senha antiga.

**Fix:** No `update` action, adicionar: `if client_params[:password].present? → merge password_plain: client_params[:password]`.
